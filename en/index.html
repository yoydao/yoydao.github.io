Youdao Dictionary screen-capture / select-translate / screenshot-OCR / hotkey fix lookup

Did Youdao Dictionary's screen word-capture (hover to look up), select-to-translate, screenshot OCR or hotkeys suddenly stop working? Get three things straight first - is it not enabled / not permitted, a specific app's compatibility (Office / PDF / browser), or the content itself (scanned image / hardware acceleration) having no readable text. Look up the symptom or keyword you hit (e.g. capture, pdf, select, hotkey, 64-bit, screenshot): first the L1 symptom & official positioning, then the L2 root cause & fix, then the rival comparison and real cases. Need install / setup? See Youdao Dictionary download / capture & select setup guide.

Step 1: split into four buckets first (L1 - don't reinstall right away)

When Youdao capture / select stops working, 9 times out of 10 it's one of four buckets - find whose fault it is first: (1) Enable / settings - Youdao isn't running in the background (tray icon greyed), or you didn't tick "enable capture / select / auto-translate on selection", or the wrong capture trigger; (2) Permissions - macOS wasn't granted "Accessibility + Screen Recording", or on Windows the target is an elevated app while Youdao isn't run as admin; (3) App compatibility - the Youdao COM add-in is disabled in Word/Excel, Acrobat protected mode blocks injection, or the browser has no capture extension; (4) Rendering / content - a scanned PDF is an image (needs OCR), the browser/PDF has GPU hardware acceleration drawing text as a texture, or mismatched multi-monitor DPI throws the popup off-screen. Enable / settingscapture . select . auto-translate . OCR power capture
Make sure Youdao runs in the background (tray icon not greyed), go to Settings -> Capture & Select and tick "enable capture", "enable select", "auto-translate on selection", and pick the right trigger (hover / hold Ctrl).
PermissionsmacOS Accessibility + Screen Recording . Windows admin / UAC
macOS: System Settings -> Privacy & Security, grant Youdao "Accessibility" and "Screen Recording"; Windows: when the target is "run as administrator", run Youdao as admin too (a hook only injects within the same privilege level).
App compatibilityOffice COM . Acrobat protected mode . 64-bit . browser extension
Office: File -> Options -> Add-ins -> COM Add-ins, tick the Youdao add-in; Acrobat: turn off "Enable protected mode at startup", and if needed copy YodaoDict.api into plug_ins; browser: install the Youdao capture extension.
Hotkeysconflict . custom . screenshot-OCR key
Settings -> Hotkeys, watch for a conflict prompt; switch to a less-clashy combo like Ctrl+Shift / Alt+letter; give screenshot OCR its own key (e.g. Ctrl+Alt+O, away from other apps).
Rendering / displayhardware acceleration . scanned files . DPI multi-monitor . fullscreen
A browser/PDF on GPU hardware acceleration renders content to a texture with no readable text (turn acceleration off); a scanned PDF is an image (use OCR); for off-screen popups on multi-monitor, tick "pin window position".
Network / installcapture speed . portable build . add-in not loaded
Online lookups slow? check the network; always install the official full build - unofficial "portable / lite" builds lack the registered capture-hook DLL, so PDF / global capture simply fails.

Step 2: the most overlooked root cause -> L2 (screen capture = global hook / DLL injection)

Key idea: screen capture doesn't "read the screen image" - Youdao injects capture code (a global hook / DLL) into the other app to read its text buffer. That explains most "mysterious" failures: (1) privilege level - a normal-rights Youdao can't inject into an app "run as administrator" (and vice versa), so "only that app won't capture" usually means mismatched privileges; (2) bitness - 32/64-bit hooks don't mix, so an old Youdao on 64-bit Acrobat throws "incompatible plug-in, can't load YodaoDict.api"; (3) sandbox - Acrobat "protected mode" is a sandbox that blocks third-party .api injection; (4) hardware acceleration - once a browser/PDF renders content to a GPU texture there's no "readable text" left to capture; (5) scanned files - an image-only PDF has no text layer, so use "OCR power capture". Real-world cases - quick read

  • Can't capture in the browser but Notepad / Word are fine: usually the browser has no Youdao capture extension, or GPU hardware acceleration rendered the page to a texture -> install the extension, or turn off "use hardware acceleration" and restart.
  • Acrobat 64-bit, selecting in a PDF does nothing, "incompatible plug-in, can't load YodaoDict.api": turn off Acrobat's Edit -> Preferences -> Security (Enhanced) -> "Enable protected mode at startup", open Youdao before Acrobat; if it persists, copy YodaoDict.api from Youdao's folder into Acrobat's plug_ins.
  • PDF text is selectable but won't capture, or the whole page won't: either it's a scanned file (image PDF with no text layer, needs "OCR power capture"), or Acrobat protected mode / an Autotray.exe lock is blocking injection.
  • After installing Office (especially Education / 365) Word can't translate a selection: the Youdao Office COM add-in is disabled -> File -> Options -> Add-ins -> COM Add-ins -> enable it; if not, reinstall Youdao to re-register the add-in.
  • After an OS update, capture and hotkeys all die on macOS: the update reset permissions -> re-grant Youdao "Accessibility" and "Screen Recording" under Privacy & Security and restart it.
  • Capture works in normal apps but fails inside one "run as administrator" app: different privilege levels mean the global hook can't inject into the elevated process -> right-click Youdao "Run as administrator".
  • Multi-monitor / high-DPI: the capture popup "flies" somewhere invisible: caused by mismatched per-screen DPI scaling -> tick "pin window position" in capture settings to nail it to the primary screen.
