有道辞書 画面取得 / 選択翻訳 / スクショ OCR / ショートカット 対処検索

有道辞書の画面取得（マウスオーバーで辞書）、選択翻訳、スクショ OCR、ショートカットが急に効かない？まず 3 つを切り分け——有効化 / 権限の問題か、特定アプリ（Office / PDF / ブラウザ）の互換性か、コンテンツ自体（スキャン / ハードウェアアクセラレーション）に読み取れる文字が無いのか。出会った症状やキーワード（例：取得、pdf、選択、ショートカット、64bit、スクショ）を検索し、まず L1 症状と公式の位置づけ、次に L2 根本原因と対処、最後に競合比較と実例を確認。導入 / 設定は 有道辞書ダウンロード / 取得・選択設定ガイド へ。

ステップ1：まず 4 分類で切り分け（L1・いきなり再インストールしない）

有道の取得 / 選択が効かない時、9 割は 4 つのどれか。まず誰のせいか切り分け：① 有効化 / 設定——有道がバックグラウンドで動いていない（トレイがグレー）、「取得 / 選択 / 選択時に自動翻訳」を入れていない、トリガーが違う；② 権限——macOS で「アクセシビリティ + 画面収録」を許可していない、Windows で対象が管理者実行アプリなのに有道は管理者でない；③ アプリ互換——Word/Excel の有道 COM アドインが無効、Acrobat 保護モードが注入を遮断、ブラウザに取得拡張が無い；④ 描画 / コンテンツ——スキャン PDF は画像（OCR が必要）、ブラウザ/PDF が GPU ハードウェアアクセラレーションで文字をテクスチャ化、マルチモニタの DPI 不一致でポップアップが画面外。詳しくは 取得・選択設定ガイド。

有効化 / 設定取得 . 選択 . 自動翻訳 . OCR 強力取得
有道がバックグラウンドで動作（トレイがグレーでない）を確認し、設定 → 取得・選択 で「取得を有効」「選択を有効」「選択時に自動翻訳」を入れ、トリガー（マウスオーバー / Ctrl 押下）を選ぶ。
権限macOS アクセシビリティ+画面収録 . Windows 管理者/UAC
macOS：システム設定 → プライバシーとセキュリティ で有道に「アクセシビリティ」「画面収録」を許可；Windows：対象が「管理者として実行」の時は有道も管理者で実行（同じ権限レベルでないとフックが注入できない）。
アプリ互換Office COM . Acrobat 保護モード . 64bit . ブラウザ拡張
Office：ファイル → オプション → アドイン → COM アドイン で有道を有効化；Acrobat：「起動時に保護モードを有効」を切り、必要なら YodaoDict.api を plug_ins へコピー；ブラウザ：有道取得拡張を入れる。
ショートカット競合 . カスタム . スクショ OCR キー
設定 → ショートカット で競合表示を確認；Ctrl+Shift / Alt+英字 のような被りにくい組合せに変更；スクショ OCR は別キー（例 Ctrl+Alt+O、他アプリと回避）。
描画 / 表示ハードウェアアクセラレーション . スキャン . DPI マルチモニタ . 全画面
ブラウザ/PDF が GPU アクセラレーションでテクスチャ化すると可視層に読める文字が無い（アクセラレーションを切る）；スキャン PDF は画像で OCR が必要；ポップアップが画面外なら「ウィンドウ位置を固定」。
ネットワーク / 導入取得速度 . 非公式版 . アドイン未読込
オンライン辞書が遅い時はネットワークを確認；必ず公式フル版を導入——非公式「ポータブル / 軽量版」は登録される取得フック DLL が無く、PDF / グローバル取得が効かない。

ステップ2：最も見落とす根因 → L2（画面取得 = グローバルフック / DLL インジェクション）

要点：画面取得は「画面画像を読む」のではなく、有道が相手アプリに取得コード（グローバルフック / DLL）を注入してその文字バッファを読む。これで「謎の」不具合の大半が説明できる——① 権限レベル：通常権限の有道は「管理者実行」アプリに注入できない（逆も）。だから「そのアプリだけ取得できない」は権限不一致が多い；② ビット数：32/64bit フックは互換が無く、旧版有道は 64bit Acrobat で「互換性のないプラグイン、YodaoDict.api を読み込めません」；③ サンドボックス：Acrobat「保護モード」はサンドボックスで第三者 .api 注入を遮断；④ ハードウェアアクセラレーション：ブラウザ/PDF が GPU でテクスチャ化すると、可視層に「読める文字」が無く取得できない；⑤ スキャン：画像 PDF は文字レイヤが無く「OCR 強力取得」が必要。下図は取得有効化セルフチェック盤、さらに下に競合比較と 2026 推計。

実例・クイック判定

  • ブラウザで取得できないが Notepad / Word は OK：多くは有道取得拡張が未導入、または GPU アクセラレーションでページがテクスチャ化 → 拡張を入れる、またはブラウザの「ハードウェアアクセラレーションを使用」をオフにして再起動。
  • Acrobat 64bit で PDF 選択が無反応、「互換性のないプラグイン、YodaoDict.api を読み込めません」：Acrobat の 編集 → 環境設定 → セキュリティ(拡張) →「起動時に保護モードを有効」を切り、先に有道→次に Acrobat；なお無理なら有道フォルダの YodaoDict.api を Acrobat の plug_ins へコピー。
  • PDF の文字は選べるのに取得できない / ページ全体で取れない：スキャン（画像 PDF・文字レイヤ無し、「OCR 強力取得」が必要）か、Acrobat 保護モード / Autotray.exe の占有で注入が遮断。
  • Office（特に教育版 / 365）導入後 Word で選択翻訳できない：有道 Office COM アドインが無効 → ファイル → オプション → アドイン → COM アドイン → 有効化；ダメなら有道を再インストールしてアドインを再登録。
  • OS 更新後、macOS で取得もショートカットも全滅：更新で権限がリセット → プライバシーとセキュリティ で有道に「アクセシビリティ」「画面収録」を再許可して再起動。
  • 通常アプリでは取得できるのに、ある「管理者実行」アプリでだけ不可：権限レベルが違い、グローバルフックが高権限プロセスに注入できない → 有道を右クリック「管理者として実行」。
  • マルチモニタ / 高 DPI で取得ポップアップが見えない位置に「飛ぶ」：各画面の DPI スケール不一致が原因 → 取得設定で「ウィンドウ位置を固定」しメイン画面左上に固定。
